# הוראה מוכנה ל-Claude Code

המשך את פרויקט WhatsPlan הקיים כאפליקציית Android אמיתית. אל תיצור פרויקט
חדש ואל תשכתב קוד שעובד ללא צורך.

## מטרה

לקמפל, לבדוק ולהפיק APK מתקין עבור Android, תוך שמירה על:

- ייבוא היסטוריית WhatsApp מקובצי TXT ו-ZIP.
- זיהוי חזרות, פגישות, הופעות, הקלטות, שינויי זמן וביטולים.
- הפרדה אמינה בין שם קבוצה, שולח ושיחה פרטית.
- מזהה קבוצה יציב המבוסס על Android Shortcut ID כשקיים.
- חיבור בין אירועים שיובאו מהיסטוריה לבין עדכונים עתידיים מהתראות.
- הצגת אישור לפני פתיחת אירוע ביומן.
- עיבוד מקומי בלבד, ללא שרת וללא העלאת תוכן שיחות.

## כללי בטיחות מחייבים

1. לפני שינוי, צור גיבוי מתוארך של התיקייה כולה ודווח את הנתיב.
2. אל תמחק קבצים, אל תריץ reset/checkout/clean ואל תשנה דבר מחוץ לפרויקט.
3. אל תוסיף Root, Accessibility Service, קריאת מסד הנתונים של WhatsApp,
   עקיפת הצפנה או API לא רשמי.
4. אל תוסיף הרשאות INTERNET, READ_CALENDAR או WRITE_CALENDAR.
5. אל תדפיס תוכן של שיחות ללוגים.
6. אם נדרשת התקנה על מכשיר פיזי או שינוי הרשאה במכשיר, עצור ובקש אישור.

## סדר עבודה

1. בצע forensic trace קצר: קרא Manifest, Gradle וכל קובצי Java והסבר את
   זרימת המידע לפני שינוי.
2. ודא שמותקנים JDK 17, Android SDK 35 וגרסת Gradle/AGP תואמת.
3. הוסף Gradle Wrapper תקני לפרויקט אם חסר.
4. הרץ את בדיקות הליבה:
   - ParserTest
   - ConversationIdentityResolverTest
5. הוסף בדיקות Android עבור:
   - EXTRA_CONVERSATION_TITLE
   - EXTRA_MESSAGES ו-MessagingStyle.Message
   - EXTRA_IS_GROUP_CONVERSATION
   - Shortcut ID יציב גם לאחר שינוי שם קבוצה
   - שיחה פרטית שאינה מסווגת כקבוצה
   - שתי קבוצות בעלות שולח זהה שאינן מתערבבות
   - עדכון מהתראה שמתחבר לאירוע שיובא מקובץ
6. קמפל debug APK. תקן רק שגיאות אמיתיות שנמצאו.
7. הרץ lint ובדיקות. אין להכריז הצלחה אם build, tests או lint נכשלו.
8. הפק:
   - app-debug.apk
   - SHA-256 של ה-APK
   - דוח קצר של הבדיקות
   - רשימת מגבלות שנותרו

## קריטריוני הצלחה

- `assembleDebug` מסתיים בהצלחה.
- כל הבדיקות עוברות.
- אין INTERNET permission.
- אין כתיבה אוטומטית ליומן.
- קבוצה, שולח ותוכן מוצגים בנפרד.
- שינוי שם קבוצה אינו יוצר זהות חדשה כאשר Shortcut ID נשאר זהה.
- עדכון או ביטול אינם יוצרים אירוע כפול.

בסיום, אל תסתפק בדיווח. הצג את הנתיב המדויק ל-APK ואת ערך ה-SHA-256 שלו.
